# Add Overview Documentation

Create comprehensive overview documentation for the yakthang orchestration system.

## Goal
Document the architecture and workflow of the yak orchestration system for new users and contributors.

## Tasks

1. Create `README.md` in project root with:
   - System overview (what is yakthang?)
   - Key components: yx, yak-box, workers, task trees
   - Quick start guide
   - Basic workflow examples
   
2. Create `docs/architecture.md` explaining:
   - Task tree structure (.yaks directory)
   - Worker lifecycle (spawn → wip → done/blocked)
   - Orchestrator vs worker roles
   - Native vs sandboxed workers
   
3. Create `docs/workflow.md` showing:
   - How to plan work (creating tasks)
   - How to spawn workers
   - How to monitor progress
   - How to handle blocked workers

## Style
- Clear, concise prose
- Code examples where helpful
- Diagrams if appropriate (ASCII art is fine)
- Focus on practical usage

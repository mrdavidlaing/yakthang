# Curate Skills from Matt Wynne's YAKS Project

Adapt high-value skills from mattwynne/yaks for yakthang orchestration.

## Background

The investigation (see `docs/investigation-mattwynne-skills.md`) identified 10 skills in Matt Wynne's yaks project. Four high-priority skills are ready for adaptation into `.openclaw/workspace/skills/` for yakthang use.

## Source Repository

**Matt Wynne's yaks:** https://github.com/mattwynne/yaks  
**Skills directory:** https://github.com/mattwynne/yaks/tree/main/src/yaks/.claude/skills

## Skills to Curate

This epic contains 4 sub-yaks, one for each skill:

### 1. yak-mapping
**Purpose:** Emergent planning through the Approach Pattern  
**Why:** Core orchestrator workflow — plan work iteratively, show map constantly  
**Status:** `curate-skills/yak-mapping`

### 2. preparing-a-yak
**Purpose:** Three-phase requirements (brainstorm → example map → plan)  
**Why:** Critical for orchestrator preparing clear work before spawning workers  
**Status:** `curate-skills/preparing-a-yak`

### 3. discovery-tree-workflow
**Purpose:** Just-in-time hierarchical task breakdown  
**Why:** Formalizes how yakthang already uses yx's tree structure  
**Status:** `curate-skills/discovery-tree-workflow`

### 4. mikado-method
**Purpose:** Discover refactoring dependencies through experiments  
**Why:** De-risk complex refactorings with test-driven discovery  
**Status:** `curate-skills/mikado-method`

## Adaptation Strategy

Each skill needs:
1. **Copy source** — bring the original SKILL.md into `.openclaw/workspace/skills/`
2. **Adapt content** — replace yaks CLI specifics with yakthang patterns
3. **Add examples** — show how to use in orchestrator + worker contexts
4. **Update AGENTS.md** — reference the new skills

## Integration

These skills work together:
- **yak-mapping** reveals what needs preparation
- **preparing-a-yak** ensures clear requirements before spawning workers
- **discovery-tree-workflow** guides when to break down vs. delegate
- **mikado-method** handles complex refactorings systematically

## Success Criteria

All 4 skills:
- Exist at `.openclaw/workspace/skills/<skill-name>/SKILL.md`
- Adapted for yakthang context (not just yaks CLI)
- Referenced in `.openclaw/workspace/AGENTS.md`
- Tested in practice (used on real yaks)

## Notes

Matt Wynne's skills are well-crafted and battle-tested. The goal is adaptation, not reinvention. Keep the core patterns; adjust the implementation details for Docker/Zellij/yak-box workflows.

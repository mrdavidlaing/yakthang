Scripted smoke test for --runtime sandbox end-to-end.

Environment: Linux (bubblewrap + socat).

Prerequisites (verify before running tests):
- srt installed: just install-srt && srt --help
- bwrap installed: which bwrap
- socat installed: which socat

## Test cases

Write as a shellspec or bash test script at src/yak-box/tests/shellspec/sandbox_smoke.sh
(or similar — match existing test conventions).

### 1. Preflight rejects missing srt
- Temporarily hide srt from PATH
- Run: yak-box spawn --runtime sandbox --cwd /tmp --yak-name preflight-test
- Assert: exit code != 0, stderr contains "srt" and "install"
- Restore PATH

### 2. srt config generation
- Call GenerateSrtConfig with a test CWD (may need a small Go test binary or
  use the existing unit tests — verify they cover the temp file creation)
- Assert: JSON is valid, contains allowWrite with the CWD, contains allowedDomains

### 3. Worker starts under sandbox
- Run: yak-box spawn --runtime sandbox --cwd <test-dir> --yak-name smoke-sandbox --tool claude "echo hello"
- Assert: worker starts (tab appears or process exists)
- Assert: worker exits cleanly (exit code 0)
- Assert: temp srt config file is cleaned up after exit

### 4. Filesystem isolation works
- Create a temp dir as CWD
- Spawn a sandbox worker that tries to:
  a) Write a file in CWD → should succeed
  b) Write a file in /tmp → should succeed (allowed)
  c) Read ~/.ssh/id_rsa (if exists) → should fail (denyRead)
- Collect exit codes / output for assertions

### 5. Network filtering works
- Spawn a sandbox worker that tries to:
  a) curl https://api.anthropic.com → should succeed (or at least connect)
  b) curl https://example.com → should fail (not in allowedDomains)
- Collect exit codes for assertions

### 6. Clean shutdown
- Spawn a sandbox worker with a long-running command
- Run: yak-box stop --yak-name <name>
- Assert: process tree terminated, temp config cleaned up, Zellij tab closed

## Notes
- Tests 3-6 require srt + bwrap + socat installed. Skip gracefully if missing.
- Use --tool claude with a simple prompt, or find a lighter test harness
  (maybe just wrap a shell command directly with srt to isolate from claude auth issues)
- For tests 4-5, it may be simpler to test srt directly rather than through
  yak-box, to isolate sandbox behavior from yak-box orchestration:
  srt --settings <config> bash -c "echo test > /tmp/test.txt"

Definition of Done:
- Smoke test script exists and is runnable
- All 6 test cases implemented (or justified skip)
- Tests pass on Linux with bwrap/socat installed
- Tests skip gracefully on machines without deps

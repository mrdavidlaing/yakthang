## What to test (through yak-box, not srt directly)

### 1. Preflight rejects missing srt
- Temporarily hide srt from PATH
- Run: yak-box spawn --runtime sandbox --cwd /tmp/test --yak-name preflight-test
- Assert: exit code != 0, stderr contains "srt" and "install"

### 2. Config generation produces correct values
- This is already covered by Go unit tests (TestGenerateSrtConfig)
- Do NOT duplicate in shellspec — just verify go test covers it

### 3. Wrapper script invokes srt correctly
- Spawn a sandbox worker, then inspect the generated run.sh wrapper script
- Assert: wrapper contains "srt --settings" with a real config path
- Assert: the srt config file exists and contains valid JSON
- Assert: config JSON contains the actual CWD in allowWrite
- Assert: config JSON contains allowedDomains (github, anthropic, etc.)
- Assert: config JSON has allowLocalBinding: true

### 4. Filesystem isolation via yak-box's generated config
- yak-box spawn --runtime sandbox --cwd <test-dir> with a command that:
  a) Writes a file in CWD → should succeed
  b) Tries to write outside CWD (e.g. /var/tmp/forbidden.txt) → should fail
- Uses yak-box's ACTUAL generated config, not a hand-crafted one

### 5. Network filtering via yak-box's generated config
- yak-box spawn --runtime sandbox with a command that:
  a) curl --max-time 5 https://api.anthropic.com → should connect (allowed)
  b) curl --max-time 5 https://example.com → should fail (not allowed)
- Uses yak-box's ACTUAL generated config

### 6. Stop cleans up srt config
- Spawn a sandbox worker with a long-running command
- Read the srt-config-path reference file from the worker home
- Run: yak-box stop --yak-name <name>
- Assert: srt config temp file is deleted
- Assert: process is terminated

# RAM Resource Profile for yak-box

## Problem
Current yak-box resource profiles (light/default/heavy) have insufficient memory for memory-intensive builds (Rust, Go, large projects):
- light: 1GB
- default: 2GB  
- heavy: 4GB

Compilation is memory-hungry, especially with:
- Large dependency trees
- Parallel compilation (cargo default uses all CPUs)
- Linking phase

Containers currently get OOM-killed during builds.

## Goals
1. Add a "ram" resource profile for high-memory workloads
2. Allow memory to use swap as safety valve (prevent OOM kills)
3. Don't constrain CPU — let builds use all available cores
4. Ensure /tmp is large enough for build artifacts

## Implementation Details

### New "ram" Profile
```go
ResourceProfile{
    Name:   "ram",
    CPUs:   "0",        // No limit — use all host CPUs
    Memory: "8g",       // 8GB RAM
    Swap:   "16g",      // 16GB swap (2x RAM)
    PIDs:   2048,       // Higher for many cargo processes
    Tmpfs: map[string]string{
        "/tmp":                    "size=4g,exec,uid=1000,gid=1000",
        "/home/yak-shaver":        "size=2g,exec,uid=1000,gid=1000",
        "/home/yak-shaver/.cache": "size=2g,exec,uid=1000,gid=1000",
    },
}
```

### Swap Support
Modify docker run to include:
- `--memory-swap=16g` (or 2x RAM) 
- Or `--memory-swap=-1` for unlimited swap

This lets containers spill to disk instead of dying.

### Cargo Job Limiter
Set `CARGO_BUILD_JOBS=4` environment variable
to reduce parallel compilation memory pressure.

## Goal

Clean up `src/yak-map` and `src/yak-box` so they feel like monorepo packages, not standalone repos. `src/yaks` stays as a submodule (upstream mattwynne/yaks).

## Current state

Both `src/yak-map` and `src/yak-box` are already tracked by the parent repo's git — no submodule extraction needed. But they still carry artifacts from when they were (or were designed as) standalone repos.

## Changes needed

### src/yak-box

1. **Delete `src/yak-box/.claude/`** — has its own `settings.local.json` and `skills/desloppify/`. Standalone repo artifact. If desloppify is useful, move it to top-level `skills/` first.
2. **Rewrite `src/yak-box/CLAUDE.md`** — currently a standalone agent guide full of OpenCode/rtk/showboat references. Replace with concise monorepo-appropriate build & test notes (Go build, shellspec tests, devcontainer purpose).
3. **Consolidate `src/yak-box/.gitignore`** — move the `yak-box` binary ignore to top-level `.gitignore` (under `src/yak-box/yak-box`). Keep `.yaks` and `.desloppify` ignores only if still relevant.
4. **Keep `src/yak-box/.devcontainer/`** — this is used at runtime by yak-box to build sandboxed worker images. It's intentional, not a standalone repo artifact.
5. **Keep `src/yak-box/.shellspec`** — test config is fine, just verify tests run from monorepo root: `cd src/yak-box && shellspec`
6. **Leave Go module path as-is** — changing `github.com/wellmaintained/yak-box` would break things for no real gain.

### src/yak-map

1. **Delete `src/yak-map/AGENTS.md`** — duplicate of CLAUDE.md content.
2. **Trim `src/yak-map/CLAUDE.md`** — keep build/test instructions, remove anything that assumes standalone repo context.

### Top-level

1. **Update top-level `.gitignore`** — add `src/yak-box/yak-box` (build output) if not already there.
2. **Verify Justfile still works** — `just build`, `just clean` should succeed after changes.

## Out of scope

- `src/yaks` submodule — leave as-is
- Go module path changes
- Moving source code between directories
- Justfile restructuring

## Acceptance criteria

- No `.claude/` directory inside `src/yak-box/`
- No `AGENTS.md` in `src/yak-map/`
- `CLAUDE.md` files in both subdirs are concise monorepo-appropriate docs
- `just build` and `just clean` still work
- Top-level `.gitignore` covers `src/yak-box/yak-box` binary
- Clean commit

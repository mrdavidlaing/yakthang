## Bug: --auto-worktree passes yak name unsanitized as git branch name

### Symptom
`yak-box spawn --auto-worktree` fails when the yak name contains spaces:

```
git worktree add <path> -b 'SC-102103 add UnixODBC manager to imogen-workstation'
fatal: 'SC-102103 add UnixODBC manager to imogen-workstation' is not a valid branch name
```

### Root cause
`--auto-worktree` passes the raw yak name as the `-b` argument to `git worktree add` without slugifying it. Git branch names cannot contain spaces.

### Expected
Sanitize the name before using it as a branch (spaces → hyphens, lowercase, strip invalid chars) — similar to how yx generates IDs.

### Workaround
Create the worktree manually before spawning:
```bash
git worktree add /tmp/my-worktree -b my-branch-name main
yak-box spawn --cwd /tmp/my-worktree ...
```

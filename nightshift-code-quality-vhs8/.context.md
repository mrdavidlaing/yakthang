Autonomous nightshift workflow for code quality improvement.

Goal: Shavers work independently on yak-map (Rust) and yak-box (Go),
using quality tooling to identify and fix issues. Each produces a PR
for morning human review.

Workflow per project:
1. Run quality checks to establish baseline
2. Identify concrete improvement opportunities
3. Implement fixes (one logical change at a time)
4. Verify all checks still pass
5. Create a PR with the improvements

Constraint: Each shaver works in a git worktree so main stays clean
and the two shavers do not conflict.

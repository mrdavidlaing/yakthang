Add change detection to yak-map's timer-driven filesystem refresh to reduce
unnecessary syscalls and improve keystroke responsiveness.

Current behavior: Every 2 seconds, the timer triggers a full recursive
walk_dir() of .yaks/ — reading every directory, sorting entries, rebuilding
the entire tree. With many yaks this causes measurable lag.

Desired behavior: Only rebuild the tree if something actually changed.

Approach options (pick the simplest that works):
A. **Modification time check**: Before walk_dir, stat the .yaks/ root dir.
   If mtime hasn't changed since last walk, skip. This catches most changes
   since adding/removing files updates parent dir mtime.
B. **Hash-based**: Hash the directory listing and compare to cached hash.
   Only rebuild if hash differs.
C. **File count check**: Count files/dirs quickly and compare to cached count.
   Rebuild only if count changed. Misses in-place edits but catches add/remove.

Option A is recommended — simplest, catches the common case.

Relevant files:
- src/yak-map/src/repository.rs (walk_dir function, lines 69-87)
- src/yak-map/src/main.rs (refresh_tasks called on timer, handle_timer)

Definition of Done:
- Timer refresh skips full tree rebuild when nothing changed
- Navigation (j/k) remains responsive with large maps (50+ yaks)
- Manual refresh ('r' key) always forces a full rebuild regardless of cache
- cargo build --target wasm32-wasip1 succeeds
- No regressions in tree rendering

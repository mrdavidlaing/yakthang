Home directories should be keyed by shaver name (--shaver-name), not task name. Also rename --name to --yak-name for clarity.

## Current behavior
- `--name` is used for: Zellij tab name, session ID, AND home directory path
- `--shaver-name` is used for: tab title decoration, assigned-to field
- Home dir: `.yak-boxes/@home/<--name>/`

## Desired behavior
- `--name` renamed to `--yak-name` → tab name / task name / session ID (what the work is called)
- `--shaver-name` → shaver identity, used for home directory (who is doing the work)
- Home dir: `.yak-boxes/@home/<--shaver-name>/`

## Why rename --name to --yak-name
Avoids confusion between the two naming concepts. `--yak-name` makes it clear this names the task/yak being worked on. `--shaver-name` names who's doing the work. The two are orthogonal.

## Why key home dirs on shaver name
OAuth credentials are stored in the shaver's home dir. If home dirs are keyed by yak name, every new task gets a fresh home dir and loses the OAuth creds. The whole point of per-shaver identity is that Yakoff's credentials persist whether he's working on 'auth-login' or 'sandbox-demo'.

## What to change
1. **Rename flag**: `--name` → `--yak-name` across all commands (spawn, stop, etc.)
   - Keep `--name` as a hidden deprecated alias if you want, but `--yak-name` is the primary
2. `sessions.EnsureHomeDir()` and `sessions.GetHomeDir()` — called with shaver name instead of yak name
3. `cmd/spawn.go` — pass shaver name to home dir functions
4. `auth-login` command — `--shaver` flag should create home at `.yak-boxes/@home/<shaver-name>/`
5. Verify preflight's `EnsureClaudeAuthEnv` receives the shaver-name-based home dir path
6. Update `--help` text for all affected commands
7. Update the orchestrator agent doc (agents/yakob.md) to use `--yak-name` instead of `--name`

- Two different yak-names with the same --shaver-name share the same home dir

- `yak-box auth-login --shaver Yakoff` creates home at `.yak-boxes/@home/Yakoff/`
- `yak-box spawn --yak-name task-xyz --shaver-name Yakoff` uses `.yak-boxes/@home/Yakoff/`
- All existing tests pass: cd src/yak-box && go test ./...

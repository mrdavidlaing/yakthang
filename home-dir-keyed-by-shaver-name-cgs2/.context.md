Home directories should be keyed by shaver name (--shaver-name), not worker/task name (--name).

## Current behavior
- `--name` is used for: Zellij tab name, session ID, AND home directory path
- `--shaver-name` is used for: tab title decoration, assigned-to field
- Home dir: `.yak-boxes/@home/<--name>/`

## Desired behavior
- `--name` → tab name / task name / session ID (what the work is called)
- `--shaver-name` → shaver identity, used for home directory (who is doing the work)
- Home dir: `.yak-boxes/@home/<--shaver-name>/`

## Why this matters
OAuth credentials are stored in the shaver's home dir. If home dirs are keyed by worker name, every new task gets a fresh home dir and loses the OAuth creds. The whole point of per-shaver identity is that Yakoff's credentials persist whether he's working on 'auth-login' or 'sandbox-demo'.

## What to change
1. `sessions.EnsureHomeDir()` and `sessions.GetHomeDir()` — called with shaver name instead of worker name
2. `cmd/spawn.go` — pass shaver name to home dir functions
3. `auth-login` command — `--shaver` flag should create home at `.yak-boxes/@home/<shaver-name>/`
4. Verify preflight's `EnsureClaudeAuthEnv` receives the shaver-name-based home dir path

## Files
- src/yak-box/cmd/spawn.go (where home dir is resolved)
- src/yak-box/internal/sessions/sessions.go (EnsureHomeDir, GetHomeDir)
- src/yak-box/cmd/auth.go (auth-login command)
- src/yak-box/internal/preflight/preflight.go (EnsureClaudeAuthEnv receives homeDir)

## Acceptance criteria
- Two different workers with the same --shaver-name share the same home dir
- OAuth creds from auth-login persist when spawning a different worker name with the same shaver
- `yak-box auth-login --shaver Yakoff` creates home at `.yak-boxes/@home/Yakoff/`
- `yak-box spawn --name task-xyz --shaver-name Yakoff` uses `.yak-boxes/@home/Yakoff/`
- All existing tests pass; cd src/yak-box && go test ./...

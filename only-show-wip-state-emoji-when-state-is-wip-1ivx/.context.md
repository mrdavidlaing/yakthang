The yak-map renderer currently shows wip-state emoji regardless of the yak's main state. 
This means a done yak with a lingering wip-state field (e.g. "ready-for-human") still 
renders with 👀🧑 instead of a clean ✓.

## Fix
In the yak-map renderer, only display wip-state emoji when the yak's main state is `wip`. 
If state is `done` or `todo`, ignore the wip-state field for rendering purposes.

## Acceptance criteria
- A done yak with wip-state set to any value renders as ✓ (no emoji)
- A todo yak with wip-state set renders as ○ (no emoji)
- A wip yak with wip-state set renders with the appropriate emoji (unchanged)
- Existing tests pass, add a test for this case

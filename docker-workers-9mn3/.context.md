Get Docker-based worker spawning working end-to-end.

Right now `spawn-worker.sh` detects Docker first (before Zellij) but fails because
`yak-worker:latest` doesn't exist and the Dockerfile has stub binaries for opencode
and yx. The goal is: `./spawn-worker.sh --cwd . --name test --task test/foo "Do the thing"`
should Just Work with Docker runtime.

## What's broken

1. **worker.Dockerfile installs stub binaries** — `opencode` and `yx` are fake
   shell scripts that just echo version strings (lines 18-25). They need to be
   real binaries.

2. **No built image** — `yak-worker:latest` has never been built, so `docker run`
   fails with "pull access denied".

3. **opencode --prompt flag** — spawn-worker.sh pipes the prompt into
   `opencode --prompt - --agent <mode>` (line 277). Need to verify this is how
   opencode actually accepts piped prompts, or fix the invocation.

## Sub-tasks

- `docker-workers/real-binaries` — Install real opencode + yx in the Dockerfile
- `docker-workers/build-and-tag` — Build the image, add a build script or Makefile target
- `docker-workers/test-e2e` — Run the same hello-yak smoke test with RUNTIME=docker

## Constraints

- Keep it minimal — ubuntu:24.04 base, only what's needed
- The image should work for the yakthang repo (bash, git, yx, opencode)
- Don't break the Zellij runtime path (it still works and should stay working)
- Worker containers should be ephemeral (--rm or cleanup-workers.sh handles it)

## Relevant files

- worker.Dockerfile — the image definition (needs real binaries)
- spawn-worker.sh — the spawn script (Docker path starts at line 222)
- check-workers.sh — monitors running containers
- cleanup-workers.sh — removes stopped containers
- kill-worker.sh — stops a specific worker

Build yak-box: a Go CLI tool for Docker-based worker orchestration.

yak-box replaces the current shell scripts (spawn-worker.sh, shutdown-worker.sh, check-workers.sh, kill-worker.sh) with a single Go binary.

Reference the existing shell scripts to understand current functionality:
- spawn-worker.sh: spawns Docker containers or Zellij tabs for workers
- shutdown-worker.sh: stops containers, clears task assignments, cleans metadata
- check-workers.sh: monitors worker status (docker ps, agent-status fields)
- kill-worker.sh: force-kills workers

Phase 1 - Scaffold:
1. Initialize a Go module in src/yak-box/
2. Set up CLI structure (cobra or similar)
3. Define subcommands: spawn, shutdown, check, kill
4. Read docs/repo-layout.md and docs/worker-spawning.md for context
5. Read the existing shell scripts to understand exact behavior
6. Create a design doc at docs/yak-box-design.md outlining the planned architecture

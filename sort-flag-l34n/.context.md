Add a --sort flag to `yx list` for controlling display order.

## Current behavior
`yx list` has no ordering control. Items appear in a fixed internal order.

## Proposed interface

  yx ls --sort <field> [--desc]

Examples:
  yx ls --sort alpha              # Alphabetical by name (default ascending)
  yx ls --sort alpha --desc       # Alphabetical descending
  yx ls --sort created            # By creation date (if tracked)
  yx ls --sort state              # Group by state (todo, wip, done)
  yx ls --sort <field-name>       # Sort by any custom field value

## Design notes
- --sort with no --desc defaults to ascending
- --desc flag reverses the sort
- Sorting by custom fields (e.g. agent-status, assigned-to) would be
  powerful for orchestrator workflows
- If a field doesn't exist on a task, sort it to the end
- Should work with all --format options (pretty, markdown, plain)
- Consider: should child tasks sort independently within their parent,
  or should sorting only apply at each tree level?

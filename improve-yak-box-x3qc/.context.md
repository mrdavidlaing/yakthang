EVERGREEN: yak-box (Go CLI) code quality improvement

This is a recurring nightshift yak. Each session, Yakob triages potential
improvements, creates sub-yaks, and spawns a shaver to work through them.

## Project details
- CWD: src/yak-box
- Language: Go
- Build check: go build && go vet ./... && go test ./...
- Integration tests: shellspec (if available)
- Desloppify state: src/yak-box/.desloppify/state-go.json
- Desloppify plan: src/yak-box/.desloppify/plan.json

## Nightshift workflow

### 1. Discovery (interactive, with operator)

Run quality checks and review desloppify issues:

```bash
cd src/yak-box
go build && go vet ./... && go test ./...    # Baseline — must pass
```

Review desloppify issues by tier:
```bash
cat .desloppify/state-go.json | jq '[.issues | to_entries[] | .value | select(.status == "open")] | group_by(.tier) | .[] | {tier: .[0].tier, count: length, examples: [.[0:3][] | .summary]}'
```

Group issues into natural clusters:
```bash
cat .desloppify/state-go.json | jq -r '[.issues | to_entries[] | .value | select(.tier == 1 and .status == "open")] | group_by(.id | split("::")[3] // .id | split("::")[2]) | .[] | "--- \(.[0].id | split("::")[2]) (\(length)) ---", (.[] | "  • \(.summary)")'
```

Look for:
- Tier 1 issues (highest priority)
- Natural clusters of related issues (same file, same pattern)
- Quick wins (low risk, mechanical changes)
- Issues safe for unattended work vs needing supervision

### 2. Curate (interactive, with operator)

Present clusters as a menu with risk assessment:

| Cluster | Issues | Risk | Nightshift safe? |
|---------|--------|------|-----------------|
| AI-debt cleanup | N | Low | ✅ Mechanical |
| Dead code removal | N | Low | ✅ Delete + verify |
| Regex hotpath | 1 | Low | ✅ Surgical |
| OAuth dedup | N | Medium | ⚠️ Cross-package |
| UnmarshalJSON | N | Medium | ⚠️ Subtle bugs |
| spawn.go refactor | N | High | ❌ Supervised only |
| Typed constants | N | Medium | ⚠️ Many files |

Operator picks which clusters go into tonight's batch.
Recommend: max 3-5 related low-risk yaks per nightshift.

### 3. Create sub-yaks

For each selected improvement, create a child yak with specific context:

```bash
yx add <improvement-name> --under improve yak-box
echo '<specific context with files, desloppify issue IDs, approach, definition of done>' | yx context <improvement-name>
```

Context must include:
- Specific files and functions to change
- Related desloppify issue IDs (for tracking)
- How to verify (go build + go vet + go test + shellspec)
- Risk level and any constraints

### 4. Spawn nightshift shaver

One shaver, all sub-yaks, in a worktree:

```bash
yak-box spawn \
  --cwd ./src/yak-box \
  --yak-name "yak-box-quality" \
  --shaver-name "<name>" \
  --tool claude --runtime native \
  --auto-worktree \
  --yaks <yak-paths...> \
  "<prompt with ordered yak list, verification steps, PR creation>"
```

Key shaver instructions:
- Work yaks sequentially in the specified order
- Run go build && go vet ./... && go test ./... after EVERY change
- Run shellspec if available
- One commit per yak (use /yak-brand)
- Create PR when all yaks done
- Do not proceed if checks fail

### 5. Morning review

- Review PR with difit: `difit --pr <url> --no-open`
- Merge or request changes
- Mark done yaks, prune, wrap
- Re-run desloppify scan to update scores if desired

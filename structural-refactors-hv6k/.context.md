Four deeper refactors. Each exercises a meatier yx pattern and takes 1-2 hours.

## Work Order (has dependencies)
1. extract tree builder — biggest readability win, domain model pattern
2. extract rendering module — display as adapter (depends on QW-1, QW-5)
3. split event handler — use case dispatch pattern
4. introduce task source trait — ports and adapters, contract testing

## Process Pattern
Each yak: use preparing-a-yak skill to spec → write ADR for the decision → implement with TDD → run bin/dev check → commit.

The preparing-a-yak workflow is the key learning here. These refactors are complex enough to benefit from speccing before coding.

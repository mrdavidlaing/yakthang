Longer-term design: how sandboxed (Docker) shavers authenticate with Claude.

## Context
yak-box supports two runtimes: native and sandboxed (Docker). Auth has two modes: API key and OAuth (Max/Pro subscription). Native runtime inherits both naturally. Sandboxed needs explicit credential propagation.

## Design decision: per-shaver OAuth identity

Each shaver name (Yakira, Yakoff, etc.) already has a persistent home directory via yak-box. The design:

1. **First spawn of a shaver**: claude login runs inside the container, prints a device-flow URL, user authorizes in browser. OAuth creds stored in the shaver's persistent home (~/.claude/).
2. **Subsequent spawns**: same home dir is mounted, OAuth creds are already there. No login needed.
3. **API key mode**: continues to work as-is via -e _ANTHROPIC_API_KEY.

### Benefits
- No host credential mounting (no read-only vs read-write dilemma)
- No token refresh races between shavers
- True isolation — each shaver has their own identity
- Maps naturally to existing persistent home dir model

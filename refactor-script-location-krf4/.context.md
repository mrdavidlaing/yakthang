# Refactor Worker Script Location

## Current Behavior

Worker scripts (run.sh, etc.) are currently written to `/tmp/worker-.../` directories.

## Desired Behavior

Now that we have persistent `.yak-boxes` folders with `@home` directories for each worker, move these scripts into the `.yak-boxes` structure instead.

## Proposed Location

Consider placing scripts in:
- `.yak-boxes/@home/<worker-name>/scripts/` - keeps them organized per worker
- Or `.yak-boxes/@home/<worker-name>/` - directly in the home folder
- Or `.yak-boxes/<worker-name>/` - separate from @home

## Benefits

1. **Persistence** - scripts survive between worker spawns
2. **Debugging** - easier to inspect worker configurations
3. **Cleanliness** - no more /tmp clutter
4. **Organization** - everything worker-related in one place

## What Needs to Change

- `yak-box spawn` command script generation logic
- Any references to `/tmp/worker-...` paths
- Documentation/comments

## Considerations

- Should scripts be cleaned up on worker stop, or persist?
- Do we need script versioning/timestamps?
- How does this interact with the `--clean` flag?

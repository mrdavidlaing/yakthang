Multi-tenant persistent Yak orchestration VM.

Architecture:
- VM in team's cloud environment with base tooling
- yakob user (non-root) runs orchestrator
- Workers run in Docker containers for isolation
- Three security layers: VM boundary, yakob user, worker containers

Security model:
- VM: Team cloud creds (read-only repo access)
- yakob user: No sudo, docker group only, git commit but no push
- Workers: Ephemeral containers, YOLO mode inside, no external creds

Key components:
1. VM provisioning script (root-only, run once)
2. Modified spawn-worker.sh (uses docker run instead of zellij tabs)
3. Worker container images/dev containers
4. Shared .yaks/ state management across containers

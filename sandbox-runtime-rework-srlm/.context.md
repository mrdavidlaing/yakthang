# Sandbox Runtime Rework

## Goal
Restructure yak-box's --runtime flag from two modes (sandboxed/native) to three
explicit modes that better reflect what each actually does.

## Design Decisions

### New runtime modes
- **--runtime native** — unchanged. Direct host process, Zellij tab, no isolation.
- **--runtime sandbox** — NEW. Uses @anthropic-ai/sandbox-runtime (open source,
  npm package from Anthropic). OS-level filesystem + network isolation via
  bubblewrap (Linux) or Seatbelt (macOS). Lighter than Docker; no container
  images or resource limits. Works with ALL tools (claude, opencode, cursor) —
  not Claude-specific.
- **--runtime devcontainer** — renamed from "sandboxed". Full Docker-based
  isolation with .devcontainer support, resource limits, read-only .yaks/ mounts,
  capability dropping, etc. Everything it does today, just a more accurate name.

### --runtime is now required (no auto-detection)
The orchestrator (Yakob) decides the isolation level when spawning workers.
Auto-detection hid this decision inside yak-box where it doesn't belong.
DetectRuntime() will be deleted entirely.

### Why sandbox-runtime works for all tools
The sandbox-runtime package wraps any command with OS-level isolation — it's
not Claude-specific. It was open-sourced by Anthropic at
https://github.com/anthropic-experimental/sandbox-runtime and can sandbox
any program (they suggest using it for MCP servers too).

### Key safety improvement
Native mode currently has ZERO protection against "shaver wipes .yaks/".
The sandbox mode will add denyWrite rules for .yaks/ (except assigned task dirs),
giving OS-level protection without Docker overhead. This fills the gap between
"full container" and "no isolation at all".

### Deprecated alias
"--runtime sandboxed" will remain as a deprecated alias for "devcontainer"
with a warning, to avoid breaking existing scripts/muscle memory.

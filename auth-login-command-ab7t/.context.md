Implement the `yak-box auth-login --shaver <name>` command from the design doc at docs/plans/2026-03-07-sandboxed-shaver-auth-design.md.

## What it does

1. Ensures the shaver's persistent home dir exists (via sessions.EnsureHomeDir)
2. Spawns a minimal Docker container with the same image and home-dir mount as a normal sandboxed worker
3. Runs `claude login` interactively inside the container
4. The device-flow URL prints to stdout; user opens it in their browser
5. Container exits after login completes; OAuth creds persist in `<homeDir>/.claude/`
6. Subsequent `yak-box spawn --shaver <name> --runtime sandboxed` picks up the creds automatically

## Docker invocation (from design doc)

```bash
docker run -it --rm \
  --name yak-auth-login-<shaver> \
  --user "<uid>:<gid>" \
  -v "<homeDir>:/home/yak-shaver:rw" \
  -e HOME=/home/yak-shaver \
  -e IS_DEMO=true \
  -v "<passwdFile>:/etc/passwd:ro" \
  -v "<groupFile>:/etc/group:ro" \
  yak-worker:latest \
  bash -c 'claude login'
```

## Implementation details

- Create `cmd/auth_login.go` with a new cobra command
- Required flag: `--shaver <name>` (the shaver identity, e.g. Yakira)
- Use `sessions.EnsureHomeDir(shaverName)` to create/get the home dir
- Generate passwd/group files the same way `generateRunScript` does in helpers.go (lines 157-169)
- The container should be interactive (`-it`) so the user can see the device flow URL
- Preflight: require Docker to be available
- No Zellij tab needed — this runs in the current terminal

Detect and recover from shavers that die without reporting back.

## Problem
The Claude Code Insights report (March 2026) documents multiple incidents:
- Yakon completed work but never wrote a shaver-message — Yakob didn't know it was done
- Yakira wiped the .yaks/ directory — catastrophic, no recovery mechanism
- Multiple sessions required manual state recovery after silent shaver deaths

## Desired Behavior
1. yak-box or Yakob should detect when a shaver has gone silent (no heartbeat, no message)
2. After a configurable timeout, flag the yak as 'blocked' with reason 'shaver unresponsive'
3. Surface the failure to Yakob immediately rather than waiting for manual discovery

## Possible Approaches
- Heartbeat file per shaver in .yaks/heartbeats/{shaver_name} — stale = dead
- yak-box polling Zellij tab status (is the tab still alive?)
- Mandatory 'I'm alive' shaver-message every N minutes (instruction in handbook skill)

## Acceptance Criteria
- Silent shaver death is detected within 10 minutes
- Yakob is notified automatically (not just on next manual yx ls)
- No false positives on shavers that are just slow

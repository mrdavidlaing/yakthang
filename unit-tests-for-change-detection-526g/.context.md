Add unit tests for the git notes hash change detection logic in yak-map.

The change detection works but has no dedicated test coverage. Add tests for:

1. **handle_timer skips rebuild when hash unchanged**
   - Set last_notes_hash to Some("abc123")
   - Mock/stub read_notes_hash to return Some("abc123")
   - Call handle_timer
   - Assert refresh_tasks was NOT called (tasks list unchanged)

2. **handle_timer triggers rebuild when hash changes**
   - Set last_notes_hash to Some("abc123")
   - Arrange for read_notes_hash to return Some("def456")
   - Call handle_timer
   - Assert refresh_tasks WAS called (tasks list updated)
   - Assert last_notes_hash is now Some("def456")

3. **handle_refresh bypasses cache**
   - Set last_notes_hash to Some("abc123")
   - Arrange for read_notes_hash to return Some("abc123") (same hash)
   - Call handle_refresh
   - Assert refresh_tasks WAS called despite same hash

4. **read_notes_hash returns None for missing file**
   - Point notes_ref_path at a nonexistent file
   - Assert read_notes_hash returns None

5. **read_notes_hash returns None for empty file**
   - Point notes_ref_path at an empty file
   - Assert read_notes_hash returns None

6. **First timer tick with no prior hash always rebuilds**
   - Set last_notes_hash to None
   - Arrange for read_notes_hash to return Some("abc123")
   - Call handle_timer
   - Assert refresh_tasks was called

Relevant files:
- src/yak-map/src/main.rs (the State struct and methods)

Note: The plugin state uses Zellij APIs, so you may need to test at the
function level (read_notes_hash) or use a test helper that constructs State
with a mock filesystem. Check how existing tests handle this.

Definition of Done:
- Tests for all 6 scenarios above
- cargo test passes (all existing + new tests)
- cargo build --target wasm32-wasip1 succeeds

Both config UnmarshalJSON methods use the Alias-type pattern that duplicates
all struct fields three times: original struct, Alias copy, manual field-copy block.

Files:
- internal/config/config.go (30+ fields)
- pkg/devcontainer/config.go (similar pattern)

This creates triple-site maintenance cost for every new field.

Fix: Use json.RawMessage, mapstructure, or embedding to avoid the Alias pattern.
Or use a simpler unmarshal approach that handles defaults without full duplication.

This is a MEDIUM RISK change — test thoroughly.
Verify: go build && go test ./... && shellspec

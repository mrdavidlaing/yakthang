In `yx show`, fields are classified as short vs long based on whether the value
contains newlines. This means `.md` fields with single-line content (like a 
one-line comments.md) render in the header alongside wip-state and shaver-message.

## Root cause
src/yaks/src/application/show_yak.rs lines 86-98 — classification uses 
`value.contains('\n')` to decide short vs long. There's no check on field name 
or extension.

## Fix
Fields with a `.md` extension should always be classified as long fields,
regardless of content length. They belong below the context block.

## Key files
- src/yaks/src/application/show_yak.rs (lines 86-98) — field classification
- src/yaks/src/adapters/user_display/mod.rs (lines 251-446) — display rendering

## Acceptance criteria
- `yx show` renders `.md` fields below the context as long-form sections
- Short fields (no `.md` extension) continue to use the newline heuristic
- `.context.md` rendering is unchanged (it's already special-cased)
- Existing tests pass, add a test for this case

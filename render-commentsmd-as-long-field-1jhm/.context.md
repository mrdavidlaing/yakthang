In `yx show`, fields with a `.md` extension (like `comments.md`) should render as long fields below the context block — similar to how `.context.md` renders — not as single-line short fields in the header.

Currently `comments.md` is crammed into the header alongside things like `shaver-message` and `wip-state`. It should appear as a separate section below the context, since it can contain multi-line content.

## Acceptance criteria
- `yx show` renders `.md` fields (e.g. `comments.md`) below the context as long-form sections
- Short fields (no `.md` extension) continue to render in the header as today
- `.context.md` rendering is unchanged

# Validate Shell-in-Container

## Goal
Test and validate that the shell-in-container functionality works correctly for yak-box workers.

## What to Test

1. **Spawn a sandboxed worker** with yak-box spawn
2. **Check the Zellij tab** - verify the shell pane is inside the container, not on the host
3. **Validate the prompt** - should show container context, not host
4. **Test basic commands** - pwd, ls, etc. should show container filesystem
5. **Check docker exec attachment** - verify docker exec is being used correctly

## How to Validate

You have native runtime access, so you can:
- Inspect the yak-box spawn code
- Test spawn a worker yourself
- Check Zellij pane configuration
- Use `docker exec` commands to verify attachment
- Look at the session state files

## Success Criteria

When spawning a sandboxed worker, the shell pane in the Zellij tab should be attached to the container via docker exec, not running on the host.

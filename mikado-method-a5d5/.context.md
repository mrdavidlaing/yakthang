## What It Provides

**Discover refactoring dependencies through experiments:**

1. Try naive change → run tests
2. Tests fail → identify blockers
3. Revert change
4. Add blockers as prerequisite yaks
5. Repeat

Creates a **Mikado graph** — dependency tree of refactoring steps discovered through experimentation.

## Why It's Valuable for Yakthang

- **Complex refactoring is risky** — this method de-risks through small experiments
- Test-driven discovery (vs. big-design-up-front)
- Perfect for workers tackling large refactorings
- Already fits yak DAG structure

## Adaptation Ideas

### For Orchestrator (Yakob)

1. **Recognize Mikado yaks**
   - Large refactorings are good candidates
   - Flag yaks that benefit from experimental approach
   - Example: "refactor authentication layer" → use Mikado

2. **Spawn experimental worker**
   - Worker's job: discover dependencies, not complete the work
   - Output: a set of prerequisite yaks to add to the tree

### For Workers

1. **Experiment loop**
   ```bash
   # Try the change
   git checkout -b experiment-refactor-auth
   # Make naive change (ignoring dependencies)
   sed -i 's/old_pattern/new_pattern/' src/**/*.rs
   # Run tests
   cargo test
   # Tests fail → document blockers
   echo "Blocker: need to update database schema" >> notes.md
   echo "Blocker: need to migrate config format" >> notes.md
   # Revert
   git checkout main
   # Create prerequisite yaks
   yx add refactor-auth/update-db-schema
   yx add refactor-auth/migrate-config
   ```

2. **Document graph**
   - Keep a `mikado-graph.md` showing discovered dependencies
   - ASCII art or mermaid diagram

3. **Iterate**
   - Work through prerequisite yaks
   - Re-run experiment to discover next layer

### Integration Points

- **With discovery-tree-workflow:** Mikado naturally creates a tree structure
- **With preparing-a-yak:** Each discovered blocker becomes a yak to prepare
- **With parallel-yak-implementation:** Independent branches of the Mikado graph can be parallelized

```
│   └── add-config-validator

```

Each node is a yak. Work bottom-up (leaves first).

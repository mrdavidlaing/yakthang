Improve how Yakob selects shaver names when spawning workers.

## Problem
Yakob's system prompt lists a static name pool but gives no guidance on rotation or randomisation. In practice Yakob defaults to the top of the list (Yakira gets everything, Bob gets nothing).

## Change Required
Update Yakob's system prompt (the yakob agent definition, wherever that lives) to:

1. **Check before picking** — before selecting a name, run:
   ```bash
   yx ls --format json | jq -r '.. | objects | select(.fields) | .fields[] | select(.[0]=="Assigned-to") | .[1]'
   ```
   (or equivalent) to discover names currently in use. Exclude those from candidates.

2. **Randomise selection** — pick from the remaining candidates using a random method, e.g.:
   ```bash
   echo -e 'Yakira\nYakoff\nYakriel\nYakueline\nYaklyn\nYakon\nYakitty\nBob' | shuf | head -1
   ```
   Combined with the in-use exclusion so the result is always an available name.

3. **Document the name pool** — keep the canonical pool list in the system prompt, clearly marked.

## Where to Make the Change
Find the Yakob agent definition / system prompt source. Likely in the yakthang repo — check:
- .claude/ agent config files
- Any AGENTS.md or yakob-specific prompt files

## Definition of Done
- Yakob system prompt updated with check-then-randomise name selection instructions
- The canonical name pool is clearly documented in that same location
- Bob has a fighting chance

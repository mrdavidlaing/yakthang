## Fix misleading --yak-path example in `yak-box spawn --help`

### Problem
The help text for `yak-box spawn` includes this example:

```
yak-box spawn --cwd ./frontend --yak-name ui-worker --mode plan --yak-path .tasks
```

The `--yak-path .tasks` is misleading — the convention is always `.yaks/`, never `.tasks/`.
This could confuse users into thinking `.tasks/` is a real alternative.

### Fix
Change the example to use `.yaks` or remove `--yak-path` from the example entirely
(since `.yaks/` is the default and rarely needs overriding). If keeping it, use a
realistic alternative like `--yak-path .yaks-staging`.

### Key files
- The `yak-box spawn` command definition — find the help/example text in the Go source
  under `src/yak-box/` (look for cobra command setup)

### Acceptance criteria
- The spawn help output no longer references `.tasks`
- Example uses `.yaks` or a realistic alternative, or the `--yak-path` flag is dropped from examples

# Persistent Worker Home Directories

Implement persistent home directories for workers tied to their personas (Yakriel, Yakov, etc.) rather than tasks.

## Goals

1. **Persistent OpenCode cost data** - don't lose session/cost information when containers stop
2. **Faster builds** - reuse cargo/rustup cache across tasks for the same worker
3. **Debugging support** - preserve bash history, logs, temp files for post-mortem
4. **Cleaner architecture** - centralized session management via `sessions.json`

### sessions.json Format

```json
{
  "compile-yx": {
    "worker": "yakriel",
    "task": "yakthang-v2/compile-yx",
    "container": "yak-shaver-compile-yx",
    "spawned_at": "2026-02-16T12:34:00Z",
    "runtime": "sandboxed",
    "cwd": "/home/yakob/yakthang"
  }
}
```

- Tracks only **active sessions**
- Entry removed on stop
- No archival/history (add later if needed)

### Key Decisions

1. **Workers have write access to `.yaks/`** - keep current behavior, workers can update their own status
2. **No session archival** - `sessions.json` only tracks active, simplify for now
3. **Home tied to persona** - not task name, enables cross-task state reuse
4. **--clean flag** - optional fresh start for a worker

## Implementation Tasks

See sub-yaks for each phase:

1. `create-sessions-package` - New Go package for sessions.json management
2. `update-spawn-logic` - Mount @home/{persona}, register session
3. `update-stop-logic` - Unregister session, keep home intact
4. `update-check-logic` - Read from sessions.json instead of .meta files
5. `add-clean-flag` - Add --clean to spawn command

## Migration

**No migration needed** - existing `.meta` files can be ignored/deleted. Clean break to new structure.

## Testing

After implementation:
1. Spawn Yakriel on compile-yx
2. Check `.yak-boxes/@home/yakriel/` exists
3. Stop worker
4. Check `sessions.json` entry removed
5. Re-spawn Yakriel on different task
6. Verify cargo cache reused (faster build)
7. Test --clean flag wipes home directory

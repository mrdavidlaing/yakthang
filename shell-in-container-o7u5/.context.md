# Shell pane should exec into the running container

## Problem
When spawning a yak-box worker, Zellij creates two panes:
1. OpenCode pane running in the container (correct)
2. Shell pane on the host filesystem (incorrect)

The shell pane should be a shell *inside* the running Docker container, not on the host.

## Goal
Make the shell pane execute `docker exec -it <container> bash` so users can interact with the container's filesystem and environment.

## Implementation
Modify the Zellij layout generation in `internal/runtime/sandboxed.go` (SpawnSandboxedWorker):

Current:
```kdl
pane size="33%" name="shell: ${CWD}"
```

Should be:
```kdl
pane size="33%" name="shell: container" {
    command "docker"
    args "exec" "-it" "${CONTAINER_NAME}" "bash"
}
```

**Challenge**: The container might not be running when the pane starts (race condition). Solutions:
- Add a retry loop wrapper script
- Wait for container to be "Up" before spawning the shell pane
- Use `docker exec` with a wait/retry

## Files to modify

- Possibly create a helper script for `docker exec` with retries

## Acceptance Criteria
- [ ] Shell pane opens a shell inside the running container
- [ ] Works reliably (handles timing/race conditions)
- [ ] Shows correct working directory (CWD) inside container
- [ ] User sees container filesystem, not host filesystem

# Add --clean Flag

Add `--clean` flag to `yak-box spawn` command to give workers a fresh start.

## Flag Definition

In `src/yak-box/cmd/spawn.go`:
```go
var spawnClean bool

func init() {
    spawnCmd.Flags().BoolVar(&spawnClean, "clean", false, 
        "Remove worker's home directory before spawning (fresh start)")
}
```

## Implementation

In `runSpawn()`, before spawning:
```go
if spawnClean {
    // Determine persona (need to resolve this before cleaning)
    persona := resolvePersona() // or whatever the logic is
    
    workerHome := filepath.Join(spawnCwd, ".yak-boxes", "@home", persona.Name)
    if err := os.RemoveAll(workerHome); err != nil {
        return fmt.Errorf("failed to clean worker home: %w", err)
    }
    fmt.Printf("Cleaned worker home: %s\n", workerHome)
}
```

## Usage

```bash
# Normal spawn - reuses Yakriel's existing home
./bin/yak-box spawn --name compile-yx --task yakthang-v2/compile-yx

# Clean spawn - removes Yakriel's home first
./bin/yak-box spawn --name compile-yx --task yakthang-v2/compile-yx --clean
```

## Help Text

Update spawn command help:
```
--clean    Give the worker a fresh start by removing their persistent
           home directory. Useful if cargo cache is corrupted or
           worker state seems confused.
```

## Testing

- Spawn worker normally, verify home directory persists
- Stop worker
- Re-spawn with --clean

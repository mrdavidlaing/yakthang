Write a .claude/skills/yak-map-architecture/ skill that codifies the conventions learned during the quality journey.

## Format
One skill file (SKILL.md) with conventions AND embedded rationale. For each convention:
- **Convention** — what to do (actionable, specific)
- **Rationale** — why it exists (so agents don't accidentally undo it)
- **When to deviate** — when the rule doesn't apply

This replaces separate ADRs — the reasoning lives alongside the guidance.

## What to Capture

### Module Structure
The 6-file layout: main.rs, model.rs, repository.rs, tree.rs, render.rs, util.rs.
What belongs where. The dependency DAG (model ← repository ← tree, model ← render, everything ← main).
Rationale: clear responsibility boundaries, no circular deps.

### Parse Once, Carry in Types
Three enums (TaskState, AgentStatusKind, ReviewStatusKind) each with FromStr/from_status_string().
String parsing at the boundary, exhaustive enum matches everywhere else.
Rationale: compiler catches missing cases, no scattered starts_with calls.
When to deviate: if the status type is only used in one place and parsing would be over-engineering.

### Named Constants Over Magic Values
ansi:: module with RED, GREEN, etc. No raw \x1b codes in implementation.
Rationale: readable, consistent, one place to change.

### Ports and Adapters (TaskSource trait)
Trait with two methods (list_tasks, get_field). Two impls: TaskRepository (filesystem), InMemoryTaskSource (tests).
Contract test macro verifies both behave identically.
Take &str for inputs (borrowed, cheap), return String for outputs (owned, caller keeps it).
Rationale: fast tests, decoupled from filesystem, behavioral parity guaranteed.
When to deviate: filesystem-specific methods (like context_path()) go on TaskRepository directly, not the trait.

### Pure Functions Over Methods
tree::build() and render functions take data in, return data out. No mutation of external state.
Rationale: testable, composable, no hidden dependencies.

### Dispatcher Pattern
update() is <20 lines routing to handle_* methods. No inline logic in match arms.
Rationale: keeps complexity scores low, each handler is independently readable.

### Quality Gate
bin/dev check runs lint, test, complexity. Thresholds: cognitive ≤ 29, cyclomatic ≤ 26.
Always run before marking done.

### Testing Patterns
- Use InMemoryTaskSource for unit tests (fast, no filesystem)
- Keep at least one filesystem smoke test per module that touches data
- Contract test macro for trait implementations
- Tests live in the module they test (#[cfg(test)] mod tests)

### WASM Build Constraints
Plugin compiles to wasm32-wasip1. Not all crates work in WASM. Test with bin/dev check which includes WASM build.

## Anti-patterns to Call Out
- starts_with() for status parsing → use the enum
- raw \x1b codes → use ansi:: constants
- inline logic in update() match arms → extract to handle_* method
- concrete TaskRepository in tests → use InMemoryTaskSource
- new functionality dumped in main.rs → find the right module

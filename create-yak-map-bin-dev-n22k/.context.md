Create a bin/dev quality gate for yak-map, modeled on src/yaks/bin/dev.

## Pattern Being Learned
**bin/dev quality gate** — yx runs clippy, rustfmt, complexity analysis, cargo audit, and mutation testing through a single entry point. This is the foundation that makes all other improvements measurable.

## What to Build
A `src/yak-map/bin/dev` script (bash) with at minimum:
- `dev check`: run all checks in sequence
- `dev lint`: clippy + rustfmt (+ cargo check for WASM target)
- `dev test`: cargo test
- `dev cx`: complexity analysis (cognitive + cyclomatic thresholds)

## Reference
Study src/yaks/bin/dev carefully — it is 978 lines. We do NOT need all of it. Start with the core: lint, test, complexity. Add mutation testing later.

## Key Constraint
yak-map compiles to WASM (wasm32-wasip1 target via Zellij). The build uses `just yakmap-build`. Make sure `dev check` exercises the WASM build path.

## Definition of Done
- `src/yak-map/bin/dev check` runs lint + test + complexity and exits 0 on current code
- Complexity thresholds are set based on current baseline (measure first, then set limits)
- Script is documented with --help
- Baseline complexity numbers recorded in yak context for future comparison

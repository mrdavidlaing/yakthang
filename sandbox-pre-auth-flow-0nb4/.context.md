Fix OAuth for sandbox workers. The problem: bubblewrap creates a separate
network namespace, so Claude's OAuth callback (localhost redirect) is
unreachable from the host browser.

Root cause: sandbox workers set CLAUDE_CONFIG_DIR=<homeDir>/.claude via
setupClaudeSettings(), but that only writes .claude.json, settings.json, etc.
It does NOT copy the host's OAuth token files. Claude then tries to re-auth
at startup but can't complete the localhost callback inside bwrap.

Native workers don't have this problem because they're on the host network.

Solution: Before wrapping with srt, copy the host's OAuth credentials into
the worker's CLAUDE_CONFIG_DIR. The host credentials live in ~/.claude/
and include .json files with OAuth tokens.

Implementation:
1. In spawn_sandbox.go, after setupClaudeSettings() but before launching srt:
   - Read the host's ~/.claude/ directory
   - Copy any .json OAuth credential files to homeDir/.claude/
   - Skip files that setupClaudeSettings already wrote (.claude.json, settings.json, etc.)
   - Alternatively: symlink the host's credential files (read-only is fine)

2. Consider a helper function copyHostOAuthCredentials(hostClaudeDir, workerClaudeDir)
   that handles the file copying with error handling.

3. srt's filesystem config already allows reads from ~ by default (only
   ~/.ssh and ~/.aws/credentials are denied). So the credential files are
   readable during spawn setup (before srt wraps).

Alternative approach: Instead of copying, just set CLAUDE_CONFIG_DIR to the
host's ~/.claude/ directly. But this breaks worker isolation — all workers
would share the same config. Copying is cleaner.

Reference: See how devcontainer does it — it mounts homeDir at /home/yak-shaver
which already has .claude/ from a prior yak-box auth-login. We're essentially
doing the same thing but with a file copy instead of a Docker volume mount.

Relevant files:
- src/yak-box/internal/runtime/spawn_sandbox.go (add copy step after setupClaudeSettings)
- src/yak-box/internal/runtime/helpers.go (reference for setupClaudeSettings)
- src/yak-box/internal/runtime/auth.go (reference for credential detection)

Definition of Done:
- Sandbox workers start with working OAuth credentials (no login prompt)
- Host OAuth credentials are copied into worker's .claude/ dir before srt launch
- go test ./... passes
- Manual test: yak-box spawn --runtime sandbox completes startup without OAuth prompt

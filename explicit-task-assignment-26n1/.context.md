Improve yak-box spawn prompt to explicitly tell workers which task(s) to focus on.

## Current behavior
When spawning with `--yaks yakthang-v2/yak-box/cleanup-zellij-tabs`, the worker gets
a generic prompt saying "work on assigned tasks" but doesn't clearly state WHICH task
to focus on.

The worker has to run `yx ls` and figure it out, which wastes tokens and time.

## Desired behavior
The spawn prompt should explicitly state:

"You are assigned to work on: <task-path>"

Or if multiple tasks:
"You are assigned to work on:
  - <task-1>
  - <task-2>"

## Implementation notes
- The prompt assembly happens in yak-box spawn code
- The --yaks flag provides the task paths
- Should be prominent in the prompt (near the top, after identity but before generic instructions)
- Should still tell them to use `yx context --show <task>` to read details
- Might also want to suggest: "Start with: yx state <task> wip"

## Acceptance criteria
- Spawned workers immediately know which task(s) they're assigned to
- No need to run `yx ls` first to figure out what to work on
- Prompt is clear and actionable

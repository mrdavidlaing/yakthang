## Overview

The devcontainer is pre-configured with everything needed to build and work with yx.

## Devcontainer Setup

The container has:
- Rust toolchain at `/opt/cargo` (root-writable)
- Pre-built yx from host at `/usr/local/bin/yx`
- Full build dependencies (openssl, libssh2, etc.)

Environment variables already configured:
```
RUSTUP_HOME=/opt/rustup
CARGO_HOME=/opt/cargo
PATH includes /opt/cargo/bin
```

## Building yx

To rebuild yx (e.g., to use unreleased features):

```bash
cd src/yaks
cargo build --release
cp target/release/yx ../bin/yx
```

Then the updated binary is available to agents in the container.

## Note

The Dockerfile copies `bin/yx` from the host at build time. Rebuild the container with `devcontainer build` to pick up any changes to the source.

Short-term fix: skip the _ANTHROPIC_API_KEY preflight check when --runtime native.

## Why
Claude Code in native mode inherits the host's OAuth session (~/.claude/) and env vars naturally. The API key preflight gate is only needed for sandboxed (Docker) workers where credentials must be explicitly passed in.

Currently EnsureClaudeAuthEnv in src/yak-box/internal/preflight/preflight.go unconditionally requires _ANTHROPIC_API_KEY for --tool claude. This blocks Max/Pro subscription users who authenticate via OAuth.

## What to change

1. **preflight.go** — Update EnsureClaudeAuthEnv to accept a runtime parameter. Skip the API key check when runtime is 'native'. Keep it required for 'sandboxed'.

2. **The caller** — Find where EnsureClaudeAuthEnv is called and pass the runtime value through.

3. **Tests** — Update preflight_test.go:
   - Existing tests should still pass for sandboxed/default runtime
   - Add test: native runtime + no API key = no error
   - Add test: native runtime + API key set = no error (still works)

Bug: yak-box stop kills the process but doesn't close the Zellij tab.

Root cause: findZellijTabIndex in native.go line 137 does exact match:
  if tab == name { return i + 1, nil }

But Zellij tab names are the DisplayName (e.g. "SmokeTest 🪒🐃 smoke-stop-test")
while name is just the yak-name (e.g. "smoke-stop-test"). Exact match fails,
returns -1, and StopNativeWorker silently returns nil.

Fix: Change line 137 from exact match to substring match:
  if strings.Contains(tab, name) { return i + 1, nil }

This matches the yak-name anywhere in the tab title, which handles both the
old format and the new "ShaverName 🪒🐃 yak-name" format.

Add a test for findZellijTabIndex with display-name-style tab names.

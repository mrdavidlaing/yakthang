When using YAK_PATH=~/Work/wellmaintained-packages-yakthang as an env var, yx creates/reads from a different git notes tree than when running from that directory directly (without the env var). This caused:

1. Session yak and worklogs written to a shadow store invisible to normal yx ls
2. Field writes (agent-status, review-verdict, etc.) landing in the wrong tree
3. Duplicate entries appearing when both stores are visible

## To investigate
- How does yx resolve YAK_PATH vs cwd-based .yaks detection?
- Are these two separate git notes refs?
- Should YAK_PATH point to the .yaks subdirectory rather than the workspace root?
- Is this a bug or user error?

## Workaround discovered
Always cd to the workspace root and run yx without YAK_PATH env var.

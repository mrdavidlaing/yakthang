## Bug: Y key clipboard copy doesn't work on macOS

### Symptom
Pressing Y in yak-map to copy a yak ID to the clipboard does nothing on macOS.
Toast message may appear but the system clipboard doesn't actually get the value.

### Root cause (likely)
`copy_via_zellij_tty()` in `src/yak-map/src/main.rs` (around line 187) uses a
shell script that:
1. Tries `pgrep -x zellij` + `/proc/$pid/fd/1` — Linux-only, doesn't exist on macOS
2. Falls back to `printf ... > /dev/tty` — may not resolve correctly in WASI context

macOS has no `/proc` filesystem, so the primary path always fails. The `/dev/tty`
fallback likely doesn't reach the actual terminal emulator.

### Key files
- `src/yak-map/src/main.rs` — all the relevant code is here
  - Line ~187: `copy_via_zellij_tty()` — the clipboard function
  - Line ~546: Y key handler that calls it
  - Line ~207: custom base64 encoding

### Acceptance criteria
- Y key copies yak ID to system clipboard on macOS
- Still works on Linux
- OSC 52 is the right approach — just needs a macOS-compatible way to find the terminal

### Hints
- On macOS, `lsof` or checking `ZELLIJ` env vars might help find the right TTY
- Alternatively, `pbcopy` is the native macOS clipboard command — could be a simpler fallback
- Check if Zellij's `run_command` has any built-in clipboard support

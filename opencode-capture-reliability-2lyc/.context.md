Investigate ways to reliably capture OpenCode worker costs even when containers don't shut down cleanly.

Current Issue:
- Workers export session data to .worker-costs/ on clean shutdown via inner.sh
- If container crashes, is killed, or exits unexpectedly, no JSON export occurs
- Yakov (yak-map-dev) ran for hours costing .25 but only has stats.txt, no JSON
- cost-summary.sh only looks at .json files, missing these uncaptured costs

Potential Solutions:
1. Periodic background export during worker operation (cron-like within container)
2. Mount opencode's native session storage and parse from there
3. Use opencode daemon/monitor mode to watch and export sessions
4. Post-hoc recovery: scan for abandoned sessions and export retroactively
5. Parse stats.txt as fallback (brittle - relies on text format)

Acceptance Criteria:
- All worker costs captured regardless of shutdown method
- No gaps in cost reporting
- Maintain current performance (don't add significant overhead)

Notes:
- Opencode sessions are stored in ~/.local/share/opencode/storage/ on host
- Worker containers may not have direct access to host paths
- Need to consider multi-worker scenario (concurrent writes)

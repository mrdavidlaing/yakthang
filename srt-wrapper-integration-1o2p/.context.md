Wire SpawnSandboxWorker to actually invoke srt wrapping the tool command.

Steps:
1. In spawn_sandbox.go, implement SpawnSandboxWorker:
   - Call GenerateSrtConfig(cwd) to get the config file path
   - Build the srt command: srt --settings <config-path> <tool-command>
   - The tool command is whatever claude/opencode/cursor invocation would
     normally run in native mode
   - Launch via exec.Command, wire stdout/stderr, wait for exit
2. On worker exit (or stop), clean up the temp config file
3. Register the sandbox worker in Zellij session tracking (same as native)

Model on the existing native runtime implementation in spawn_native.go —
sandbox is essentially native + srt wrapper.

Relevant files:
- src/yak-box/internal/runtime/spawn_sandbox.go (fill in the stub)
- src/yak-box/internal/runtime/spawn_native.go (reference implementation)

Definition of Done:
- yak-box spawn --runtime sandbox --cwd . --yak-name test --tool claude
  launches claude wrapped in srt with the generated config
- Worker appears in Zellij tab
- Stopping the worker cleans up the srt config temp file
- go test ./... passes

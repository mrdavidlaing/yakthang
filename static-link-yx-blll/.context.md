yx is dynamically linked against libgit2.so.1.9, libllhttp.so.9.3, and libssh2.so.1 — none of which exist in the Ubuntu 24.04 container (apt only has libgit2 1.7). 

Options explored:
1. apt-get install libgit2-dev — version mismatch (1.7 vs 1.9 needed)
2. Copy .so files from host into build context — works but fragile, requires staging before every docker build
3. Static linking — proper fix, not attempted yet

Recommendation: build yx with static libgit2 linking (cargo feature flag or vendored build). Until then, sandboxed workers cannot use yx — they need to read .yaks/ files directly.

yx is dynamically linked against libgit2.so.1.9 which doesn't exist in the Ubuntu 24.04 container (only 1.7 available via apt). Currently we copy the .so from the host as a workaround. Proper fix: build yx with static linking so it works in any container without needing the shared lib. Alternatively, vendor libgit2 1.9 in the Dockerfile build.

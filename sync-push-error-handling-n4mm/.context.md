Improve yx sync error handling and user feedback.

## Problems
1. **Silent push failures** - push_to_remote() uses 'let _ = remote.push(...)' which ignores errors
2. **No feedback** - When push fails (no write access, network issue), yx doesn't tell the user
3. **Sync still 'succeeds'** - Command returns Ok(()) even if push failed

## Solution
- Report push failures to the user with actionable error messages
- Distinguish between 'sync succeeded locally' vs 'sync succeeded and pushed to remote'
- Provide guidance when push fails (e.g., 'Run: git push origin refs/notes/yaks:refs/notes/yaks')

## Context
User discovered yaks weren't syncing to new machine because initial push silently failed. Had to manually run 'git push origin refs/notes/yaks:refs/notes/yaks' to fix.

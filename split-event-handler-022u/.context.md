Break the 73-line update() event handler into focused methods.

## Pattern Being Learned
**UseCase pattern** — yx uses one struct per command (src/yaks/src/application/). Each has execute(). CLI routing in main.rs dispatches to the right use case. For yak-map, we dont need full use cases, but splitting the match arms into named methods is the same principle.

**Preparing-a-yak** — Spec this with the preparing-a-yak skill before implementing.

## What to Extract
update() (lines 500-572) handles:
- Navigation (up/down/page-up/page-down/home/end)
- Edit action (enter key → open in yx)
- Copy action (c key → clipboard)
- Show action (s key → show details)
- Refresh (timer tick)
- Resize

## Target
update() becomes a dispatcher:
  match event {
    Key(Up|Down|...) => self.handle_navigation(key),
    Key(Enter) => self.handle_edit(),
    Key(c) => self.handle_copy(),
    ...
  }

## Definition of Done
- update() is <20 lines (pure dispatch)
- Each handler is a separate method with clear name
- All tests pass
- No behavior change

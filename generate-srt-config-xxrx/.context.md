Build the srt JSON config file for a sandbox worker.

Hardcoded config to start — no per-spawn customisation yet.

Config structure (from spike findings):
```json
{
  "filesystem": {
    "allowWrite": ["<cwd>", "/tmp"],
    "denyWrite": [],
    "denyRead": ["~/.ssh", "~/.aws/credentials"],
    "allowRead": []
  },
  "network": {
    "allowedDomains": [
      "github.com", "*.github.com",
      "api.anthropic.com", "*.anthropic.com",
      "registry.npmjs.org",
      "crates.io", "static.crates.io",
      "proxy.golang.org", "sum.golang.org"
    ],
    "deniedDomains": []
  }
}
```

Steps:
1. Create internal/runtime/srt_config.go with a GenerateSrtConfig(cwd string) function
2. Takes the worker CWD, returns the JSON config as a temp file path
3. Expand ~ in paths to actual $HOME
4. Write to a temp file that gets cleaned up after worker exits

Relevant files:
- src/yak-box/internal/runtime/srt_config.go (new)
- src/yak-box/internal/runtime/srt_config_test.go (new)

Definition of Done:
- GenerateSrtConfig("/home/user/project") returns valid JSON config path
- Config allows writes to CWD and /tmp
- Config allows network to standard dev domains
- Temp file is created in os.TempDir()
- go test ./... passes

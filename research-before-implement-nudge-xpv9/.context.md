Add a 'research before implement' nudge to shaver-facing skills.

## Problem
The Claude Code Insights report shows 26 'wrong approach' friction events — the
#1 friction type. Claude frequently jumps to implementation with incorrect or
deprecated syntax before researching the correct approach. Examples:
- Hyprland gestures: tried deprecated syntax, then concluded feature was removed,
  before finding the correct bind-based syntax after multiple rounds
- SBOM scoring: implemented wrong formula before checking existing code
- Config fixes: fixed one error at a time instead of validating fully

## Desired Changes
1. yak-shaving-handbook skill: add explicit instruction — 'Before implementing,
   verify your approach. Check docs, existing code, or run --help. If unsure,
   say so in shaver-message rather than guessing.'
2. yak-mapping skill: when writing task context, include a 'research first'
   note for tasks involving unfamiliar APIs or config syntax
3. yak-sniff-test skill: flag the 'serial fix commits' anti-pattern — if the
   shaver made 3+ commits fixing the same file, note it in review

## Acceptance Criteria
- Shaver skills explicitly discourage guess-and-check implementation
- Sniff test catches the pattern when it happens anyway
- Measurable: fewer 'wrong approach' friction events in next report

## Tool Evaluation: Leash vs packnplay vs yak-box

**Context:** David Laing requested deep comparison of [Leash](https://github.com/strongdm/leash) and [packnplay](https://github.com/obra/packnplay) vs yak-box for yakthang orchestration system.

**Key Findings:**
- **Leash:** Security-focused container wrapper with Cedar policy engine, syscall monitoring, MCP inspection. Strong on security, weak on orchestration.
- **packnplay:** Lightweight container/worktree launcher with automatic git worktree management, DevContainer spec support, credential handling. Strong on isolation/reproducibility.
- **yak-box:** Unique orchestration layer with DAG task tracking (yx), multi-worker coordination, visual monitoring (YakMap).

**Decision:** Don't adopt either wholesale. Borrow selectively:
1. **Priority 1:** Integrate packnplay's automatic worktree management (critical for "git worktree yak" vision)
2. **Priority 2:** DevContainer spec support from packnplay
3. **Optional:** Leash's Cedar policies for security-conscious mode

**Technical Win:** Both packnplay and yak-box are written in Go → enables direct code integration, no language barriers.

**Implementation Path:**
```bash
# Phase 1: Add to yak-box spawn
yak-box spawn --task auth/api --auto-worktree
# → Auto-creates: ~/.local/share/yakthang/worktrees/api/auth-api
# → Mounts worktree into container
# → Updates yx task metadata
```

**Core Value Preservation:** Keep yak-box's unique orchestration capabilities (yx DAG, multi-worker coordination, YakMap visualization, worker status protocol). Neither alternative provides this.

## OpenClaw Compaction Configuration for 128KB Context Models

**Problem:** Default "safeguard" compaction mode waits too long before compacting on models with smaller context windows (128KB/~32k tokens).

**Solution:** Adjust `agents.defaults.compaction` settings for more aggressive, predictable compaction:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "mode": "default",
        "reserveTokensFloor": 4000,
        "maxHistoryShare": 0.5,
        "memoryFlush": {
          "enabled": true,
          "softThresholdTokens": 20000
        }
      }
    }
  }
}
```

**Key Settings:**
- `mode: "default"` → Compacts earlier instead of waiting for near-overflow
- `reserveTokensFloor: 4000` → Reserves minimum 4k tokens for model output
- `maxHistoryShare: 0.5` → Limits history to 50% of context (leaves room for system prompt + future turns)
- `memoryFlush.softThresholdTokens: 20000` → Flushes durable memories when context exceeds 20k tokens

**Token Budget for 32k Model:**
- ~12k for output buffer
- ~16k for conversation history  
- Remainder for system prompt + bootstrap context

**Apply:** `openclaw config patch --patch compaction.json` or `/config` command in chat

## GitHub Copilot Model Context: 128K Not 200K

**Correction:** GitHub Copilot Claude models (including claude-sonnet-4.5) have **128K token context**, not 200K like native Claude API models.

**Impact:** Requires different compaction settings than initially recommended.

**Updated Settings for 128K Context:**

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "mode": "default",
        "reserveTokensFloor": 8000,
        "maxHistoryShare": 0.6,
        "memoryFlush": {
          "enabled": true,
          "softThresholdTokens": 90000
        }
      }
    }
  }
}
```

**Token Budget (128K total):**
- ~8K reserved for output buffer
- ~77K for history (60% of 128K via maxHistoryShare)
- ~43K for system prompt + workspace context
- Flush memories at 90K tokens (~70% capacity)

**Compaction Behavior Explained:**
- **Memory Flush (Stage 1):** At 90K tokens, pause and extract durable facts to `memory/YYYY-MM-DD.md`
- **Compaction (Stage 2):** When history exceeds 77K tokens, summarize older turns + keep recent ~10-20 messages intact
- **Mode difference:** `"default"` compacts proactively at threshold vs `"safeguard"` waits until ~95% full

**Manual Memory Flush:** Send message like "Save memories" or "Pre-compaction memory flush" to trigger immediate flush without waiting for threshold.

**Task Created:** `yakthang-v2/openclaw-config/github-copilot-compaction` to apply these settings via `gateway config.patch`

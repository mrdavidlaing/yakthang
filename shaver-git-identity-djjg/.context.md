Ensure shavers use correct git user.name and user.email when making commits.

## Problem
When a shaver (spawned via yak-box) makes a git commit, it's unclear what identity it uses. It should either:
1. Mirror the host's global git config (user.name / user.email), or
2. Use a per-shaver identity like `user+<shaver-name>@domain` for attribution

## Investigation needed
- What git identity do native-runtime shavers currently use when committing?
- What git identity do sandboxed-runtime shavers currently use?
- How does yak-box spawn set up the git environment? Check prompt.go, native spawner, and Dockerfile.
- Does the yak-brand skill (commit skill) set Co-Authored-By but leave the author as-is?

## Options to evaluate
1. **Pass through host git config** — simplest, shaver commits look like human commits
2. **Per-shaver identity** — e.g. `David Laing (via Yakira) <david+yakira@example.com>` — gives traceability
3. **Hybrid** — use host identity but add shaver name in Co-Authored-By or commit trailer

## Acceptance criteria
- Document what currently happens (both runtimes)
- Pick an approach and implement it
- Verify commits from shavers have the correct identity

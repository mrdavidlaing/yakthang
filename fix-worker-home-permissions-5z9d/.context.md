# Fix Worker Home Permissions

## Problem

Worker home directories at `.yak-boxes/@home/{persona}/` contain root-owned subdirectories that can't be cleaned up without sudo:

```
.yak-boxes/@home/Yakriel/.local/    (root:root)
.yak-boxes/@home/Yakueline/.local/  (root:root)
```

These were created by Docker containers when OpenCode tried to write to the mounted home.

## Required Action

Run these commands to clean up:

```bash
cd /home/yakob/yakthang
sudo rm -rf .yak-boxes/@home/*/
```

Or fix ownership:

```bash
sudo chown -R yakob:yakob .yak-boxes/@home/
```

## Prevention

After cleanup, the spawn code has been modified to pre-create `.local` directories with correct permissions before mounting.

## Verification

After cleanup:
```bash
ls -la .yak-boxes/@home/    # Should be empty or yakob-owned
./bin/yak-box spawn --cwd . --name test-perm --task test-task-1
docker ps | grep test-perm   # Should show running container
```

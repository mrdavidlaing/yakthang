## Task
Raise a GitHub issue against mattwynne/yaks proposing additional native yak states to reduce reliance on custom fields.

## Background
Currently yx has 3 states: todo, wip, done. Yakthang uses 13 custom fields to fill gaps. Several of these fields duplicate or extend what states should handle natively.

## Research Summary (already completed)

### Current custom fields in use:

**Communication fields (agent‚Üîorchestrator):**
- `agent-status` ‚Äî wip:/blocked:/done: prefixes. This is the BIG one. The `blocked` prefix has no native state equivalent. A native `blocked` state would let yx ls show blocked yaks distinctly and let tooling react without parsing field values.
- `yakob-message` ‚Äî Yakob‚Üíagent push channel (polled by agent). This is messaging, not state ‚Äî likely stays as a field.
- `comments.md` ‚Äî Detailed notes from agent. Also messaging ‚Äî stays as a field.

**Review workflow fields:**
- `review-status` ‚Äî in-progress/pass/fail. yak-map already renders emoji for these. A native `review` state (or `done` sub-states) could replace this.
- `review-verdict` ‚Äî pass:/fail:/needs-info: summary. Complements review-status.
- `review-notes` ‚Äî Detailed review findings on fail.

**Metadata fields:**
- `assigned-to` ‚Äî Worker assignment. Set by yak-box spawn. Could be a first-class yx concept.
- `spend` ‚Äî Cumulative cost tracking. Stays as field.
- `task-cost` ‚Äî Per-task cost. Stays as field.
- `worktrees` ‚Äî Multi-repo coordination. Stays as field.

**Session fields (on session yaks only):**
- `hard-stop`, `wip-limit`, `started` ‚Äî Session constraints. These are metadata on a special yak type; fields are appropriate.

## Proposed New States

Based on the analysis, these native states would reduce field overhead:

1. **`blocked`** ‚Äî Most impactful. Currently encoded as `agent-status: blocked: <reason>`. A native state would:
   - Show distinctly in `yx ls` (different bullet/color)
   - Enable `yx ls --state blocked` filtering
   - Remove the need to parse agent-status field for blocked detection
   - Yakob could react to state changes rather than polling field values

2. **`review`** ‚Äî Currently encoded as `review-status: in-progress`. A native state between done and truly-done would:
   - Make the review gate visible in `yx ls`
   - Replace the review-status field for the in-progress case
   - pass/fail could transition to done or back to wip

3. **`parked`** ‚Äî Not currently a field, but a common need. Yaks that are intentionally set aside (not blocked, not todo ‚Äî deliberately deferred). Currently these just sit as `todo` with no distinction.

## What stays as fields
- Communication: yakob-message, comments.md, agent-status (even with blocked state, the wip:/done: prefixes carry useful detail)
- Cost: spend, task-cost
- Metadata: assigned-to, worktrees, hard-stop, wip-limit, started
- Review detail: review-verdict, review-notes (review-status could be replaced by state)
